DDL: Conceitos

O que é DDL?

Data Definition Language: linguagem para definir estruturas

Define esquema do banco: tabelas, índices, views, constraints

Operações DDL executam autocommit por padrão no MySQL

Comandos principais

CREATE: criar banco, tabela, índice, view, procedure

ALTER: modificar estrutura existente

DROP: remover objeto do banco definitivamente

TRUNCATE: remover todos os dados, mantendo estrutura

Definição de esquema

Define a estrutura do banco de dados

Estabelece constraints: chaves primárias, estrangeiras, unicidade

Relações entre tabelas via foreign keys

Boas práticas

Use sempre charset utf8mb4 e collation consistente

Cuidado com DDL em produção (autocommit, locks)

Nomes claros e consistentes para tabelas e colunas

Defina tipos de dados adequados para cada coluna

DDL modifica o dicionário de dados. Comandos DDL podem ser versionados em scripts de migração para evolução controlada do esquema.
