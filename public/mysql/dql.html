DQL: Conceitos do SELECT

O que é SELECT?

Principal comando DQL para consultar dados no banco

Retorna um conjunto de linhas (result set) temporário

Altamente flexível: filtros, junções, agrupamentos, ordenação

Principais Cláusulas

FROM: Define tabela(s) de onde serão selecionados os dados

WHERE: Filtra linhas com base em condições

GROUP BY: Agrupa registros com base em colunas

HAVING: Filtra grupos após agregação

ORDER BY: Ordena resultados por colunas específicas

LIMIT: Restringe o número de linhas retornadas

Ordem de Execução

O MySQL processa na ordem:

1 FROM + JOINs

2 WHERE

3 GROUP BY

4 HAVING

5 SELECT

6 ORDER BY

7 LIMIT

Entender a ordem de execução é fundamental para otimizar consultas complexas e resolver problemas de performance em queries SQL.
