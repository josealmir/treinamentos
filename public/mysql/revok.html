DCL: REVOKE (revogar permissões)

Exemplos e dicas para revogar permissões de usuários no MySQL:

-- Remover permissão específica de um usuário

REVOKE UPDATE ON biblioteca.loans FROM 'autorenew'@'localhost';

-- Remover todas as permissões e opção de concessão

REVOKE ALL PRIVILEGES, GRANT OPTION FROM 'userx'@'localhost';

-- Revogação parcial (MySQL 8.0.16+)

SET PERSIST partial_revokes = ON;

-- Conceder privilégios amplos mas restringir banco específico

GRANT ALL PRIVILEGES ON *.* TO 'admin'@'localhost';

REVOKE ALL PRIVILEGES ON mysql.* FROM 'admin'@'localhost';

GRANT SELECT ON mysql.* TO 'admin'@'localhost';

-- Verificar permissões após revogar

SHOW GRANTS FOR 'admin'@'localhost';

Considerações de segurança:

  • Sempre verifique com SHOW GRANTS após revogar permissões
  • Cuidado especial com WITH GRANT OPTION - usuários podem ter concedido permissões a outros
  • A revogação parcial permite políticas refinadas de acesso (necessário MySQL 8.0.16+)
  • Em ambientes de produção, priorize o princípio do menor privilégio
  • Mantenha um log de alterações de permissões para auditoria
