DML: Conceitos

O que é DML?

Data Manipulation Language - Linguagem de manipulação de dados

INSERT: adiciona novos registros às tabelas

UPDATE: modifica registros existentes nas tabelas

DELETE: remove registros das tabelas

Transações em DML

Use START TRANSACTION para agrupar operações

COMMIT para confirmar mudanças

ROLLBACK para desfazer alterações em caso de erro

Garantem consistência e atomicidade das operações

Integridade Referencial

Foreign Keys (FK) garantem consistência entre tabelas

Ações de FK: CASCADE, RESTRICT, SET NULL

Evitam dados órfãos e violações de referência

Boas Práticas

Sempre usar WHERE em UPDATE/DELETE

Testar DML em ambiente de desenvolvimento

Considerar soft delete em vez de DELETE físico

Usar prepared statements contra SQL injection

DML é a parte do SQL que permite modificar dados em tabelas previamente criadas com DDL. Diferente do DDL, operações DML são transacionais no InnoDB.
