DCL: Conceitos e Escopos

O que é DCL?

Data Control Language: gerencia segurança e acesso ao banco

Comandos principais: GRANT (concede) e REVOKE (revoga)

Gerencia usuários, permissões e controle de acesso

Escopos de Privilégios

Global (*.*): aplicado ao servidor inteiro

Banco de dados (database.*): aplicado a um BD específico

Tabela (database.table): aplicado a uma tabela

Coluna: aplicado a colunas específicas de uma tabela

Princípio do Menor Privilégio

Conceder apenas as permissões mínimas necessárias

Exemplo: usuário de relatórios precisa apenas de SELECT

Tipos de Privilégios

Estáticos: padrão, como SELECT, UPDATE, DELETE

Dinâmicos: plugins/componentes específicos (MySQL 8+)

Roles: conjunto de privilégios reutilizáveis (MySQL 8+)

GRANT OPTION

Permite conceder suas permissões a outros usuários

Privilégio poderoso e potencialmente perigoso

Use com moderação; geralmente só para DBAs

Tarefas Comuns

Criar e gerenciar usuários

Atribuir permissões granulares

Auditar permissões com SHOW GRANTS

No MySQL 8, o sistema de privilégios foi remodelado para melhor segurança, separando privilégios estáticos e dinâmicos e introduzindo roles.
