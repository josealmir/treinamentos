DML: UPDATE

Exemplos de como atualizar registros em tabelas com diferentes níveis de complexidade:

-- Atualização simples de registros
UPDATE pedidos SET total = total + 10.00
WHERE pedido_id = 1001;

-- Atualização com múltiplas colunas
UPDATE clientes
SET email = 'novo.email@exemplo.com',
    telefone = '(11) 99999-8888'
WHERE cliente_id = 42;

-- UPDATE com JOIN (aplicar desconto para clientes premium)
UPDATE pedidos p
JOIN clientes c ON c.cliente_id = p.cliente_id
SET p.total = p.total * 0.9 -- aplica 10% de desconto
WHERE c.tipo = 'premium'
AND p.data_criacao >= '2025-01-01';

-- UPDATE dentro de uma transação (recomendado)
START TRANSACTION;
UPDATE contas SET saldo = saldo - 500
WHERE conta_id = 101;
UPDATE contas SET saldo = saldo + 500
WHERE conta_id = 202;
COMMIT;

Cuidados ao usar UPDATE:

  • Sempre use a cláusula WHERE para evitar atualizar todos os registros acidentalmente
  • Use transações para garantir consistência em operações complexas
  • Considere criar um backup ou testar o comando com SELECT antes de executar
  • JOINs em UPDATEs podem melhorar a performance vs subconsultas
  • Atualizações em lote (quando necessárias) devem considerar o tamanho do binlog
