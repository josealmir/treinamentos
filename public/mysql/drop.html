DDL: DROP vs TRUNCATE

Característica DROP TABLE TRUNCATE TABLE
Definição

Remove completamente a tabela e sua definição

Remove todos os dados, mantém a estrutura
Velocidade

Geralmente mais rápido

Mais rápido que DELETE, similar a DROP
Auto Increment

Elimina a sequência completamente

Reset para o valor inicial (normalmente 1)
Registro

Não registra remoções individuais

Não registra remoções individuais
Transação

Não pode ser revertido com ROLLBACK

Comportamento varia conforme o SGBD

Exemplos de Código

-- Remover tabela completamente

DROP TABLE produtos;

-- Remover tabela se existir

DROP TABLE IF EXISTS produtos;

-- Remover todos os dados mantendo estrutura

TRUNCATE TABLE produtos;

Quando Usar

DROP TABLE

  • Quando a tabela não é mais necessária

  • Para remover definições/estruturas antigas

  • Durante redefinição completa de esquemas

TRUNCATE TABLE

  • Apagar todos os dados para reuso da tabela

  • Para "resetar" uma tabela mantendo índices

  • Em ambientes de teste para limpeza rápida

Cuidado!

Tanto DROP quanto TRUNCATE são irreversíveis em contexto normal. Se precisar de rollback, considere usar DELETE com WHERE e dentro de uma transação.
