DML: INSERT

Exemplos de inserção de dados em tabelas usando diferentes variações do comando INSERT:

-- Inserção simples (uma linha)
INSERT INTO clientes (nome_completo, email)
VALUES ('Ana Lima', 'ana@exemplo.com');

-- Inserção de múltiplas linhas
INSERT INTO clientes (nome_completo, email)
VALUES
('Bruno Dias', 'bruno@exemplo.com'),
('Carla Santos', 'carla@exemplo.com'),
('Daniel Oliveira', 'daniel@exemplo.com');

-- INSERT com SELECT (copiar dados entre tabelas)
INSERT INTO pedidos (cliente_id, total)
SELECT c.cliente_id, 0
FROM clientes c
WHERE c.email LIKE '%@exemplo.com';

-- UPSERT (INSERT ... ON DUPLICATE KEY UPDATE)
INSERT INTO clientes (nome_completo, email, telefone)
VALUES ('Ana Lima', 'ana@exemplo.com', '(11) 98765-4321')
ON DUPLICATE KEY UPDATE
nome_completo = VALUES(nome_completo),
telefone = VALUES(telefone);

Dicas importantes:

  • Para melhor performance, agrupe múltiplas inserções em um único comando INSERT
  • Use INSERT IGNORE para ignorar erros como duplicidade de chaves
  • INSERT ... SELECT é eficiente para copiar grandes volumes de dados
  • Use UPSERT (ON DUPLICATE KEY UPDATE) quando não souber se o registro já existe
