DML: DELETE

Exemplos e segurança ao remover dados de tabelas:

-- Remover linhas específicas com filtro WHERE
DELETE FROM pedidos
WHERE criado_em < '2024-01-01';

-- DELETE com JOIN (remover pedidos sem clientes válidos)
DELETE p FROM pedidos p
LEFT JOIN clientes c ON c.cliente_id = p.cliente_id
WHERE c.cliente_id IS NULL;

-- Usando transação para segurança
START TRANSACTION;
DELETE FROM itens_pedido WHERE pedido_id = 1001;
DELETE FROM pedidos WHERE pedido_id = 1001;
-- ROLLBACK; -- descartar alterações
COMMIT; -- confirmar alterações

-- Alternativa: soft delete (exclusão lógica)
ALTER TABLE clientes ADD COLUMN ativo TINYINT(1) DEFAULT 1;
UPDATE clientes SET ativo = 0 WHERE cliente_id = 10;

-- Usar WHERE sempre! Sem WHERE apaga TUDO:
-- DELETE FROM pedidos; -- PERIGO! Remove todos os dados

Práticas de segurança:

  • Sempre use cláusula WHERE específica para evitar remoção acidental de dados
  • Use transações (START TRANSACTION) para operações em várias tabelas
  • Considere soft delete para dados importantes (flag ativo/inativo)
  • Backup antes de operações DELETE em massa
  • Use LIMIT para restringir número de linhas removidas em operações grandes
