SQL por Categorias

DDL (Data Definition Language)

  • Define a estrutura do banco de dados: tabelas, índices, constraints
  • CREATE: criar bancos de dados, tabelas e outros objetos
  • ALTER: modificar a estrutura de objetos existentes
  • DROP: remover objetos completamente
  • TRUNCATE: limpar dados mantendo estrutura

DML (Data Manipulation Language)

  • Manipula os dados dentro das tabelas
  • INSERT: adicionar novas linhas
  • UPDATE: modificar dados existentes
  • DELETE: remover registros

DQL (Data Query Language)

  • Realiza consultas para recuperação de dados
  • SELECT: principal comando para consultas
  • Cláusulas: FROM, WHERE, GROUP BY, HAVING, ORDER BY, LIMIT

DCL (Data Control Language)

  • Controla permissões e acesso aos dados
  • GRANT: concede privilégios a usuários
  • REVOKE: remove privilégios de usuários

TCL (Transaction Control Language)

  • Controla transações no banco de dados
  • COMMIT: confirma alterações
  • ROLLBACK: desfaz alterações
  • SAVEPOINT: cria pontos de restauração

Algumas fontes consideram o comando SELECT como parte do DML, não classificando DQL como uma categoria separada.
