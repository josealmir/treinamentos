Async/Await e Tratamento de Erros

Async/await é uma forma moderna e mais legível de trabalhar com Promises e operações assíncronas em JavaScript.

Vantagens do Async/Await

  • Código mais limpo e legível
  • Parece código síncrono (mais intuitivo)
  • Facilita o tratamento de erros
  • Melhor encadeamento de operações

Tratamento de Erros

  • Use blocos try/catch para capturar erros
  • Trate erros de rede e da API
  • Verifique o status da resposta (ex: 404, 500)
  • Implemente retry ou fallback quando necessário

Exemplo de Código:

// Função assíncrona com tratamento de erros
async function buscarDados(userId) {
  try {
    // Aguarda a resposta da requisição
    const resposta = await fetch(
      `https://api.exemplo.com/usuarios/${userId}`
    );
    
    // Verifica se a requisição foi bem-sucedida
    if (!resposta.ok) {
      throw new Error(`Erro HTTP: ${resposta.status}`);
    }
    
    // Converte a resposta para JSON
    const dados = await resposta.json();
    return dados;
    
  } catch (erro) {
    console.error('Falha na requisição:', erro);
    // Podemos tratar diferentes tipos de erros aqui
    throw erro; // Propaga o erro para quem chamou
  }
}

Sempre inclua tratamento de erros ao trabalhar com requisições. Interfaces de usuário devem fornecer feedback quando algo falha.
