Gerenciamento de Dependências: npm e package.json

O npm (Node Package Manager) é o gerenciador de pacotes padrão para o Node.js, permitindo o compartilhamento e reutilização de código JavaScript.

O arquivo package.json

  • Manifesto do projeto e suas dependências
  • Metadados: nome, versão, descrição
  • Lista de dependências e suas versões
  • Scripts de automação de tarefas

Comandos npm essenciais

  • npm init: Cria um novo package.json
  • npm install [pacote]: Instala dependências
  • npm install --save-dev [pacote]: Instala devDependencies
  • npm run [script]: Executa scripts definidos

Exemplo de package.json:

{
  "name": "meu-projeto-javascript",
  "version": "1.0.0",
  "description": "Aplicação web com JavaScript",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "test": "jest"
  },
  "dependencies": {
    "express": "^4.18.2",
    "axios": "^1.3.0"
  },
  "devDependencies": {
    "nodemon": "^2.0.22",
    "jest": "^29.5.0"
  }
}

Versionamento: O caractere ^ significa que o npm pode instalar versões mais recentes compatíveis (ex: ^4.18.2 permite qualquer versão 4.x.x maior que 4.18.2).
