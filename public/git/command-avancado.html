Comandos Avançados: Stash e Rebase

Git Stash

Salva temporariamente alterações que você ainda não quer fazer commit, permitindo alternar entre tarefas sem perder o trabalho em andamento.

Quando usar:

  • Quando precisa mudar de branch rapidamente
  • Para interromper trabalho em progresso
  • Quando recebe uma tarefa urgente
  • Para "limpar" o workspace temporariamente

Comandos básicos:

Salvar alterações no stash:

git stash

Listar itens no stash:

git stash list

Aplicar o stash mais recente:

git stash apply

Aplicar um stash específico:

git stash apply stash@{2}

Aplicar e remover do stash:

git stash pop

Git Rebase

Reescreve o histórico de commits, aplicando commits de uma branch em cima de outra, criando um histórico linear e mais limpo.

Quando usar:

  • Para manter histórico limpo e linear
  • Antes de enviar pull/merge requests
  • Para incorporar mudanças da branch principal
  • Para reescrever/combinar seus commits

Exemplo de uso:

git checkout feature
git rebase master

Aplica os commits da branch feature sobre a master

Cuidado!

Nunca use rebase em commits já publicados/compartilhados! Isso reescreve a história e pode causar problemas para outros desenvolvedores.

Rebase interativo:

git rebase -i HEAD~3

Permite editar, combinar ou reordenar os últimos 3 commits
