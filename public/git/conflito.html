Resolução de Conflitos

O que são conflitos?

Conflitos ocorrem quando o Git não consegue resolver automaticamente diferenças no código entre dois commits.

Quando ocorrem conflitos?

  • Duas pessoas modificam a mesma linha de código
  • Uma pessoa edita um arquivo enquanto outra o exclui
  • Durante um merge, rebase ou pull
  • Ao tentar fazer checkout para outra branch

Como identificar conflitos

O Git marca os conflitos no código com marcadores especiais:

<<<<<<< HEAD
Suas alterações (na branch atual)
=======
Alterações de outro desenvolvedor
>>>>>>> branch-name

O Git também mostra mensagens no terminal alertando sobre conflitos.

Como resolver conflitos

1

Identificar os arquivos com conflito

Use git status para listar os arquivos com conflito

2

Editar os arquivos manualmente

Abra cada arquivo, remova os marcadores de conflito e decida qual código manter

3

Marcar como resolvido

Adicione os arquivos corrigidos com git add <arquivo>

4

Concluir o processo

Continue o processo com git merge --continue ou git rebase --continue

Ferramentas úteis

Use ferramentas visuais para facilitar a resolução:
• git mergetool - inicia ferramenta configurada
• IDEs modernas (VS Code, IntelliJ) têm resolução integrada
