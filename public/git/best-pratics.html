Git

Boas Práticas com Git

Práticas de Desenvolvimento

Commits pequenos e descritivos

Faça commits frequentes e específicos para cada alteração lógica do código

Use branches adequadamente

Crie branches separados para cada funcionalidade ou correção

Sincronização frequente

Faça pull regularmente do repositório remoto para evitar conflitos grandes

Use .gitignore corretamente

Configure arquivos que não devem ser versionados (logs, dependências, etc.)

Mensagens de commit eficientes

Use o formato: "[tipo]: descrição curta (até 50 caracteres)"
Exemplo: "feat: adiciona autenticação de usuários"

Colaboração e Segurança

Pull Requests

Use PRs para revisão de código antes de integrar ao branch principal

Versione com Tags

Use tags para marcar releases e versões importantes do projeto

Resolva Conflitos

Resolva conflitos logo que surgirem para evitar problemas maiores

Documente

Mantenha README atualizado com instruções de instalação e uso

O que evitar:

  • Fazer commit diretamente na branch main/master
  • Mensagens de commit vagas como "corrigido" ou "atualizado"
  • Commits com muitos arquivos e alterações diversas
  • Compartilhar credenciais ou dados sensíveis no código
