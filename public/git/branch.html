Branches: Desenvolvendo em Paralelo

O que são Branches?

Branches são ramificações independentes do código que permitem trabalhar em funcionalidades, correções ou experimentos sem afetar a linha principal de desenvolvimento (master/main).

Comandos Principais

Criar uma nova branch:

git branch nome-da-branch

Criar e mudar para a nova branch:

git checkout -b nome-da-branch

Listar todas as branches:

git branch

Mudar para outra branch:

git checkout nome-da-branch

Excluir uma branch:

git branch -d nome-da-branch

Exemplo Prático

Cenário: Nova funcionalidade de login

1. Criar branch para a funcionalidade

git checkout -b feature/login

2. Fazer alterações e commits na branch

git add .
git commit -m "Implementa tela de login"

3. Voltar para a branch principal

git checkout main

4. Mesclar as alterações (quando concluídas)

git merge feature/login

Boas Práticas

Nomenclatura

Use prefixos como feature/, bugfix/, hotfix/

Escopo

Foco em uma única tarefa por branch

Atualização

Sincronize com a branch principal

Duração

Evite branches de longa duração

Dica!

Use git branch -v para ver o último commit de cada branch e git branch --merged para ver quais branches já foram mescladas.
