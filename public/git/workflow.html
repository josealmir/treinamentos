Workflow do Git: Como Funciona na Prática

O Git mantém seu código em três áreas principais. Entender este fluxo é essencial para usar o Git efetivamente.

Working Directory

Onde você modifica seus arquivos

git add

Staging Area

Arquivos preparados para o próximo commit

git commit

Repository

Histórico de commits do projeto

Working Directory

  • • Contém todos os arquivos do projeto
  • • Onde você edita e cria arquivos
  • • git status mostra arquivos modificados

Staging Area

  • • Área de preparação para o próximo commit
  • • git add . adiciona todos os arquivos
  • • git reset remove arquivos do staging

Repository

  • • Contém todos os commits do projeto
  • • git log mostra o histórico
  • • git push envia commits para remoto

Dica Prática

Use git diff para ver mudanças no Working Directory e git diff --staged para ver o que está na Staging Area.
