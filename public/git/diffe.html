Git, GitHub e GitLab: Diferenças

Embora frequentemente mencionados juntos, Git, GitHub e GitLab têm funções diferentes no ecossistema de desenvolvimento. Entenda as diferenças e quando usar cada um:

Git

Sistema de controle de versão distribuído

  • Software local instalado no computador
  • Gerencia histórico de alterações no código
  • Criado por Linus Torvalds em 2005
  • Funciona independentemente de servidores

GitHub

Plataforma de hospedagem de repositórios Git

  • Serviço online para armazenar repositórios
  • Focado em open-source e colaboração
  • Adquirido pela Microsoft em 2018
  • Oferece GitHub Actions para CI/CD

GitLab

Plataforma DevOps completa com Git

  • Serviço online ou auto-hospedado
  • Forte foco em CI/CD integrado
  • Enfatiza segurança e privacidade
  • Oferece versão open-source e comercial

Lembre-se:

O Git é a tecnologia de base, enquanto GitHub e GitLab são serviços baseados nela. Você precisa do Git para usar qualquer um deles, mas não precisa do GitHub ou GitLab para usar o Git.
