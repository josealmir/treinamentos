Comandos de Colaboração: clone, pull, push

Principais Comandos para Colaboração

Estes comandos permitem trabalhar com repositórios remotos, essenciais para projetos colaborativos e compartilhamento de código.

git clone

Cria uma cópia completa de um repositório Git remoto em sua máquina local.

git clone https://github.com/usuario/repositorio.git

git pull

Atualiza seu repositório local com as mudanças do repositório remoto.

git pull origin main

git push

Envia seus commits locais para o repositório remoto.

git push origin feature-branch

Fluxo de Trabalho Colaborativo

  1. Comece com um clone

    Obtenha uma cópia completa do repositório para começar a trabalhar.

  2. Pull antes de modificar

    Sempre atualize seu repositório local antes de começar novas alterações.

  3. Faça commits das alterações

    Registre suas mudanças em commits pequenos e descritivos.

  4. Push para compartilhar

    Envie seus commits para o repositório remoto quando estiver pronto.

Dica importante:

O comando git pull é na verdade uma combinação de dois comandos: git fetch (baixa alterações) + git merge (integra alterações).

Cuidado!

Sempre faça um git pull antes de um git push para evitar conflitos de merge.
