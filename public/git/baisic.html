Conceitos Básicos: Repositório, Commit, Branch

O Git se baseia em três conceitos fundamentais que você precisa entender para utilizá-lo efetivamente:

Repositório

Local onde o Git armazena todos os arquivos e histórico do seu projeto.

  • Pode ser local (na sua máquina) ou remoto (GitHub, GitLab)
  • Contém todo o histórico de alterações
  • Criado com o comando git init
$ git init meu-projeto

Commit

Captura instantânea do estado atual dos arquivos monitorados.

  • Registra o que foi alterado e por quem
  • Inclui uma mensagem descritiva
  • Cada commit tem um identificador único (hash)
$ git commit -m "Adiciona recurso X"

Branch

Linha de desenvolvimento independente do código principal.

  • Permite desenvolver recursos sem afetar o código principal
  • Facilita o trabalho em equipe (cada dev em sua branch)
  • O branch principal é chamado master ou main
$ git branch nova-feature

Dica importante:

Esses três conceitos trabalham juntos: você cria commits em um branch específico, dentro de um repositório. Entender como eles se relacionam é essencial para dominar o Git!
