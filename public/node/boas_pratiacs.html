Boas Práticas e Performance

Checklist de otimização e boas práticas

Schemas e Validação

  • Defina schemas em TODAS as rotas Body, params, query e response
  • Use TypeBox para validação Tipagem e validação centralizadas
  • Valide inputs antes de processar Prevenção de injeção de dados

Banco de Dados

  • Pool de conexões MySQL Evite criar conexões por requisição
  • Índices em queries críticas Use EXPLAIN para análise

Segurança

  • Rate limiting @fastify/rate-limit
  • CORS configurado @fastify/cors
  • Variáveis de ambiente validadas dotenv + zod

Logging

  • Logging estruturado Pino com requestId
  • Logs de erro detalhados Stack traces e contexto

Observabilidade

  • Métricas e healthcheck Monitoramento de saúde
  • Tracing distribuído Rastreamento de requisições

Automação

  • Migrações automáticas Prisma/Knex
  • CI/CD para lint/test/build Pipeline automatizado
